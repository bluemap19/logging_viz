## 一、模块概述

### 1.1 模块定位

表格数据管理模块负责**离散型地质数据**的加载、处理和管理。这类数据通常以深度区间或离散深度点的形式记录岩性、沉积相、含油气性等地层属性信息，是测井解释和地质建模的重要基础数据。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| **数据加载** | 读取 CSV/XLSX 格式的表格数据 |
| **格式转换** | 支持 Type2(单深度点) 和 Type3(深度区间) 格式 |
| **类别编码** | 自动将文本类别映射为整数编码 |
| **数据融合** | 与测井曲线数据深度对齐和合并 |

### 1.3 技术特点

- 支持两种数据格式：深度区间 (Type3) 和单深度点 (Type2)
- 自动生成类别编码映射规则
- 支持自定义类别替换字典
- 支持深度展开 (区间→离散点)
- 支持与测井数据无缝合并

### 1.4 应用场景

- 岩性柱状图绘制
- 沉积相分析
- 测井解释模型训练
- 地层对比
- 储层分类评价

### 2.2 数据文件格式

#### Type3 格式 (深度区间)

**命名规则：** `{井名}__LITHO_TYPE.csv` 或包含 `LITHO_TYPE` 关键字

**文件结构：**
| 列名 | 类型 | 说明 |
|------|------|------|
| depth_start | float | 区间起始深度 (m) |
| depth_end | float | 区间结束深度 (m) |
| type | str/int | 岩性/类别名称 |

**示例数据：**
```csv
depth_start,depth_end,type
3200.5,3210.2,砂岩
3210.2,3225.0,泥岩
3225.0,3230.5,砂砾岩
3230.5,3245.8,泥质粉砂岩
```

**特点：**
- 紧凑存储，适合大段连续相同岩性
- 需要展开才能与测井曲线对齐
- 深度区间必须连续且无重叠

#### Type2 格式 (单深度点)

**文件结构：**
| 列名 | 类型 | 说明 |
|------|------|------|
| depth | float | 深度 (m) |
| type | str/int | 岩性/类别名称 |

**示例数据：**
```csv
depth,type
3200.0,砂岩
3201.0,砂岩
3202.0,泥岩
3203.0,泥岩
3204.0,砂砾岩
```

**特点：**
- 与测井曲线格式一致
- 可直接与测井数据合并
- 数据量较大

### 3.3 核心方法详解

#### load_data()

**目的：** 从 CSV 或 XLSX 文件加载表格数据

**原理：**
1. 检测文件扩展名
2. 使用相应方法读取文件
3. 自动检测数据格式 (Type2 或 Type3)
4. 提取类别信息并生成映射

**代码示例：**
```python
def load_data(self):
    import pandas as pd
    import os
    
    file_ext = os.path.splitext(self.path)[1].lower()
    
    if file_ext == '.csv':
        self.data = pd.read_csv(self.path)
    elif file_ext in ['.xlsx', '.xls']:
        self.data = pd.read_excel(self.path)
    else:
        raise ValueError(f"Unsupported file format: {file_ext}")
    
    # 检测数据格式
    self.data_type = self.detect_format()
    
    # 生成类别映射
    self._generate_category_mapping()
```

#### detect_format()

**目的：** 自动检测数据格式类型

**原理：**
- 检查列名是否包含 `depth_start` 和 `depth_end` → Type3
- 检查列名是否包含 `depth` (单深度) → Type2

**代码示例：**
```python
def detect_format(self) -> str:
    columns = self.data.columns.str.lower()
    
    if 'depth_start' in columns and 'depth_end' in columns:
        self.data_type = 'type3'
        return 'type3'
    elif 'depth' in columns:
        self.data_type = 'type2'
        return 'type2'
    else:
        raise ValueError("Cannot detect data format. Expected columns: "
                        "['depth_start', 'depth_end', 'type'] or ['depth', 'type']")
```

#### get_data(mode, replaced, replace_dict, new_col)

**目的：** 获取处理后的表格数据

**参数：**
- `mode`: 输出格式模式 ('2' 或 '3')
- `replaced`: 是否进行类别替换/编码
- `replace_dict`: 自定义类别替换字典
- `new_col`: 输出列名

**返回：**
- 处理后的 DataFrame

**代码示例：**
```python
def get_data(self, mode='3', replaced=True, replace_dict=None, new_col='Lithology'):
    result = self.data.copy()
    
    # 格式转换
    if mode == '2' and self.data_type == 'type3':
        # 需要展开到深度点 (需要目标深度数组)
        raise ValueError("Mode '2' requires target depth array for expansion")
    
    # 类别替换/编码
    if replaced:
        if replace_dict:
            # 使用自定义替换字典
            result[new_col] = result['type'].map(replace_dict)
        else:
            # 使用自动生成的映射
            result[new_col] = result['type'].map(self.category_mapping)
    
    return result
```

#### expand_to_depth_points(target_depth)

**目的：** 将 Type3 深度区间格式展开到指定深度点

**参数：**
- `target_depth`: 目标深度数组 (通常来自测井数据)

**返回：**
- 展开后的 DataFrame，包含 depth 和 type 列

**原理：**
1. 遍历每个目标深度点
2. 找到该深度所属的区间
3. 将区间 type 赋值给该深度点

**代码示例：**
```python
def expand_to_depth_points(self, target_depth: np.ndarray) -> pd.DataFrame:
    import numpy as np
    
    if self.data_type != 'type3':
        return self.data.copy()
    
    # 创建结果数组
    expanded_type = np.full(len(target_depth), np.nan, dtype=object)
    
    # 对每个深度点，找到所属区间
    for i, depth in enumerate(target_depth):
        mask = (self.data['depth_start'] <= depth) & (depth < self.data['depth_end'])
        if mask.any():
            expanded_type[i] = self.data.loc[mask, 'type'].iloc[0]
    
    # 创建结果 DataFrame
    result = pd.DataFrame({
        'depth': target_depth,
        'type': expanded_type
    })
    
    return result
```

#### get_category_mapping()

**目的：** 获取类别到整数的映射字典

**原理：**
1. 统计所有唯一类别
2. 按字母顺序或出现频率排序
3. 分配从 0 开始的整数编码

**代码示例：**
```python
def _generate_category_mapping(self):
    # 获取所有唯一类别
    self.categories = sorted(self.data['type'].unique().tolist())
    
    # 生成映射
    self.category_mapping = {cat: idx for idx, cat in enumerate(self.categories)}
    self.reverse_mapping = {idx: cat for cat, idx in self.category_mapping.items()}
    
    return self.category_mapping

def get_category_mapping(self) -> Dict[str, int]:
    return self.category_mapping
```

#### apply_mapping(data, col_name)

**目的：** 将类别映射应用到数据

**代码示例：**
```python
def apply_mapping(self, data: pd.DataFrame, col_name: str = 'type') -> pd.DataFrame:
    result = data.copy()
    result[f'{col_name}_encoded'] = result[col_name].map(self.category_mapping)
    return result
```

```python
              key: str = '',

              replaced: bool = True,

    """
```

### 4.4 测井 - 表格数据合并

```python
def combine_logging_table(self,
                          logging_key: str,
                          curve_names_logging: List[str],
                          table_key: str,
                          replace_dict: Dict = None,
                          new_col: str = 'Lithology',
                          norm: bool = False):
    """
    合并测井数据与表格数据
    
    参数:
        logging_key: 测井文件路径关键字
        curve_names_logging: 测井曲线名称列表
        table_key: 表格文件路径关键字
        replace_dict: 类别替换字典
        new_col: 新列名
        norm: 是否归一化测井数据
    
    返回:
        合并后的 DataFrame
    """
    # 获取测井数据
    logging_data = self.get_logging(
        key=logging_key,
        curve_names=curve_names_logging,
        norm=norm
    )
    
    # 获取表格数据并展开到测井深度
    table_obj = self.table_dict.get(table_key)
    if table_obj and table_obj.data_type == 'type3':
        table_expanded = table_obj.expand_to_depth_points(logging_data['DEPTH'].values)
    else:
        table_data = self.get_table(key=table_key, replaced=True, 
                                    replace_dict=replace_dict, new_col=new_col)
        # 插值对齐到测井深度
        from scipy.interpolate import interp1d
        # ... 插值逻辑
    
    # 合并
    merged = logging_data.merge(table_expanded, on='depth', how='left')
    
    return merged
```

## 八、类别编码策略

### 8.1 自动编码规则

**默认行为：**
1. 统计所有唯一类别
2. 按字母顺序排序
3. 从 0 开始分配整数编码

**示例：**
```python
categories = ['泥岩', '砂岩', '砂砾岩', '灰岩']
# 排序后：['泥岩', '灰岩', '砂岩', '砂砾岩']
# 编码：{'泥岩': 0, '灰岩': 1, '砂岩': 2, '砂砾岩': 3}
```

### 8.2 自定义编码建议

**地质意义编码：**
```python
# 按沉积粒度编码
replace_dict = {
    '泥岩': 0,        # 细粒
    '粉砂岩': 1,      # 中细粒
    '砂岩': 2,        # 中粒
    '砂砾岩': 3,      # 粗粒
    '砾岩': 4         # 粗粒

```

**储层质量编码：**
```python

    '泥岩': 0,        # 非储层
    '泥质砂岩': 1,    # 差储层
    '砂岩': 2,        # 中储层
```

### 8.3 编码注意事项

1. **保持一致性：** 多井对比时使用相同编码规则

## 九、常见问题

### Q1: Type3 格式展开后深度点不匹配？

**原因：** 测井深度点可能超出表格深度区间范围

**解决方案：**
```python
# 检查深度范围
logging_depth_min = logging_data['DEPTH'].min()
logging_depth_max = logging_data['DEPTH'].max()
table_depth_min = table_data['depth_start'].min()
table_depth_max = table_data['depth_end'].max()

# 如果测井深度超出表格范围，进行截断或外推
if logging_depth_min < table_depth_min:
    logging_data = logging_data[logging_data['DEPTH'] >= table_depth_min]
if logging_depth_max > table_depth_max:
    logging_data = logging_data[logging_data['DEPTH'] <= table_depth_max]
```

### Q2: 类别映射后出现 NaN？

**原因：** 表格中存在未定义的类别

**解决方案：**
```python
# 检查未映射的类别
unmapped = table_data[~table_data['type'].isin(replace_dict.keys())]['type'].unique()
print(f"未映射的类别：{unmapped}")

# 添加默认映射
replace_dict_default = {cat: -1 for cat in unmapped}  # -1 表示未知
replace_dict.update(replace_dict_default)
```

### Q3: 深度区间有重叠或间隙？

**解决方案：**
```python
from src_table.table_process import fix_depth_gaps

# 自动修复深度间隙
table_fixed = fix_depth_gaps(table_data, method='extend')
```
