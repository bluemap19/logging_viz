## 一、模块概述

### 1.1 模块定位

单井数据管理模块是整个测井数据可视化系统的**核心数据管理层**，负责单井所有类型测井数据的统一扫描、识别、加载和管理。它作为数据类型层 (DataLogging/DataFMI/DataTable/DataNMR) 的上层封装，提供统一的数据访问接口，是连接文件系统和上层应用 (项目管理、可视化) 的关键枢纽。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| **文件扫描** | 自动扫描井文件夹，识别各类测井文件 |
| **数据分类** | 基于关键字自动分类文件到对应数据类型 |
| **统一接口** | 提供四类数据的统一访问接口 |
| **懒加载** | 按需加载数据，提高内存效率 |
| **数据合并** | 支持测井数据与表格数据的合并 |
| **井信息摘要** | 生成单井数据摘要信息 |

### 1.3 技术特点

- 模块化设计，四类数据独立管理
- 基于关键字的文件自动识别
- 懒加载策略，优化内存使用
- 统一的数据访问模式
- 支持多文件管理和搜索

### 1.4 系统架构位置


## 三、核心方法详解

### 3.1 文件扫描 (scan_files)

**目的：** 扫描井文件夹，识别并分类所有测井相关文件

**原理：**
1. 遍历井文件夹中的所有文件
2. 根据文件扩展名和文件名关键字进行分类
3. 将文件路径存储到对应的路径列表

**代码示例：**
```python
def scan_files(self):
    """扫描井文件夹中的所有测井文件"""
    from src_file_op.dir_operation import search_files_by_criteria
    
    # 扫描测井文件
    self.path_list_logging = search_files_by_criteria(
        self.well_path,
        name_keywords=self.LOGGING_KW,
        file_extensions=['.xlsx', '.csv'],
        all_keywords=False
    )
    
    # 扫描表格文件
    self.path_list_table = search_files_by_criteria(
        self.well_path,
        name_keywords=self.TABLE_KW,
        file_extensions=['.xlsx', '.csv'],
        all_keywords=False
    )
    
    # 扫描 FMI 文件
    self.path_list_fmi = search_files_by_criteria(
        self.well_path,
        name_keywords=self.FMI_KW,
```

### 3.2 懒加载机制

**设计思想：**
- 不在初始化时加载所有数据
- 仅在首次访问时加载对应数据
- 已加载的数据对象缓存在字典中

**实现模式：**
```python
def init_logging(self, path: str = ''):
    """初始化测井数据对象 (懒加载)"""
    if path not in self.logging_dict:
        # 确定目标路径
        target_path = path if path else (self.path_list_logging[0] if self.path_list_logging else '')
        
        if target_path:
            # 创建并缓存数据对象
            from src_well_data.data_logging_normal import DataLogging
            self.logging_dict[target_path] = DataLogging(
                path=target_path,
                well_name=self.WELL_NAME
            )

def get_logging(self, key: str = '', curve_names: List[str] = None,
                norm: bool = False, depth_limit: List[float] = None):
    """获取测井数据"""
    # 确保数据已加载
    self.init_logging(key)
    
    # 获取对象
    obj = self._get_default_obj(self.logging_dict, key)
    
    # 返回数据
    return obj.get_data(curve_names, depth_limit, norm) if obj else None
```

### 3.3 通用对象获取 (_get_default_obj)

**目的：** 从字典中获取对象，支持空 key 时返回第一个对象

**代码示例：**
```python
def _get_default_obj(self, data_dict: Dict, key: str):
    """
    获取字典中的默认对象
    
    参数:
        data_dict: 数据对象字典
        key: 查找关键字 (路径)
    
    返回:
        数据对象或 None
    """
    if key and key in data_dict:
        return data_dict[key]
    elif data_dict:
        # 返回第一个对象
        return next(iter(data_dict.values()))
    return None
```

### 3.4 文件搜索 (_search_path_list)

**目的：** 在路径列表中搜索包含特定关键字的文件

**代码示例：**
```python
def _search_path_list(self, base_list: List[str], new_kw: List[str]) -> List[str]:
    """
    搜索路径列表的通用方法
    
    参数:
        base_list: 基础路径列表
        new_kw: 额外关键字列表
    
    返回:
        匹配的路径列表
    """
    if not new_kw:
        return base_list
    
    result = []
    for path in base_list:
        # 检查路径是否包含所有关键字
        if all(kw in path for kw in new_kw):
            result.append(path)
    
    return result
```

### 3.5 数据合并 (combine_logging_table)

**目的：** 将测井曲线数据与表格数据 (如岩性) 合并

**原理：**
1. 获取测井数据
2. 获取表格数据并展开到测井深度
3. 基于深度列进行合并

**代码示例：**
```python
def combine_logging_table(self,
                          logging_key: str,
                          curve_names_logging: List[str],
                          table_key: str,
                          replace_dict: Dict = None,
                          new_col: str = 'Lithology',
                          norm: bool = False):
    """
    合并测井数据与表格数据
    
    参数:
        logging_key: 测井文件路径关键字
        curve_names_logging: 测井曲线名称列表
        table_key: 表格文件路径关键字
        replace_dict: 类别替换字典
        new_col: 新列名
        norm: 是否归一化测井数据
    
    返回:
        合并后的 DataFrame
    """
    # 获取测井数据
    logging_data = self.get_logging(
        key=logging_key,
        curve_names=curve_names_logging,
        norm=norm
    )
    
    if logging_data is None:
        return None
    
    # 获取表格数据对象
    self.init_table(table_key)
    table_obj = self._get_default_obj(self.table_dict, table_key)
    
    if table_obj is None:
        return logging_data
    
    # 如果表格是 Type3 格式，展开到测井深度
    if table_obj.data_type == 'type3':
        table_expanded = table_obj.expand_to_depth_points(
            logging_data['DEPTH'].values
        )
        
        # 应用类别替换
        if replace_dict:
            table_expanded[new_col] = table_expanded['type'].map(replace_dict)
        else:
            table_expanded[new_col] = table_expanded['type'].map(
                table_obj.category_mapping
            )
        
        # 合并
        merged = logging_data.merge(
            table_expanded[['depth', new_col]],
            left_on='DEPTH',
            right_on='depth',
            how='left'
        )
        merged = merged.drop(columns=['depth'])
    else:
        # Type2 格式，直接合并
        table_data = self.get_table(
            key=table_key,
            replaced=True,
            replace_dict=replace_dict,
            new_col=new_col
        )
        
        merged = logging_data.merge(
            table_data,
            on='DEPTH',
            how='left'
        )
    
    return merged
```

### 3.6 井信息摘要 (well_summary)

**目的：** 生成单井数据的摘要信息

**代码示例：**
```python
def well_summary(self) -> Dict:
    """
    获取井信息摘要
    
    返回:
        包含井信息的字典
    """
    summary = {
        'well_name': self.WELL_NAME,
        'well_path': self.well_path,
        'file_counts': {
            'logging': len(self.path_list_logging),
            'table': len(self.path_list_table),
            'fmi': len(self.path_list_fmi),
            'nmr': len(self.path_list_nmr)
        },
        'data_objects': {
            'logging': len(self.logging_dict),
            'table': len(self.table_dict),
            'fmi': len(self.FMI_dict),
            'nmr': len(self.NMR_dict)
        }
    }
    
    # 添加深度范围信息
    if self.path_list_logging:
        logging_obj = self._get_default_obj(self.logging_dict, '')
        if logging_obj and logging_obj.data is not None:
            summary['depth_range'] = {
                'min': float(logging_obj.data['DEPTH'].min()),
                'max': float(logging_obj.data['DEPTH'].max())
            }
    
    return summary
```

## 七、性能优化

### 7.1 懒加载策略

**优势：**
- 减少初始化时间
- 降低内存占用
- 按需加载，避免不必要的数据读取

**实现：**
```python
# 只在首次访问时加载
def get_logging(self, key: str = '', ...):
    self.init_logging(key)  # 检查并加载
    obj = self._get_default_obj(self.logging_dict, key)
    return obj.get_data(...) if obj else None
```

### 7.2 对象缓存

**机制：**
- 已加载的数据对象存储在字典中
- 后续访问直接返回缓存对象
- 避免重复加载同一文件
