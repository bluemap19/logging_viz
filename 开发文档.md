## 一、项目概述

### 1.1 项目定位

**专业测井数据可视化系统**，支持多类型测井数据的统一管理与可视化展示。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| **多数据类型** | 常规测井、电成像 (FMI)、离散表格、核磁 (NMR) |
| **文件自动识别** | 基于关键字自动扫描和分类文件 |
| **统一数据管理** | 单井数据统一封装，多井项目统一管理 |
| **交互式可视化** | 滑动条控制、深度导航、自适应布局 |
| **可扩展架构** | 模块化设计，易于添加新数据类型 |

### 1.3 技术栈

```
Python 3.8
├── 数据处理
│   ├── pandas (数据操作)
│   └── numpy (数值计算)
├── 可视化
│   ├── matplotlib (绘图引擎)
│   └── 自定义可视化类
├── 文件操作
│   └── 自定义文件扫描模块
└── 数据处理
    └── 自定义数据处理模块
```

### 3.1 LOGGING_PROJECT (项目管理类)

**位置：** `src_well_project/LOGGING_PROJECT.py`

**职责：** 管理多井项目，提供统一的数据访问接口

#### 核心属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `PROJECT_PATH` | str | 项目根路径 |
| `WELL_NAMES` | List[str] | 井名列表 |
| `WELL_PATH` | Dict[str, str] | 井名→路径映射 |

#### 核心方法

```python
class LOGGING_PROJECT:
    def __init__(self, project_path: str)
    def init_well_path(self)  # 初始化井路径
    def get_well_data(self, well_name, file_path, curve_names, Norm)  # 获取测井数据
    def get_well_data_by_charters(self, well_name, target_path_feature, target_file_type, curve_names, Norm)  # 按特征获取数据
```

#### 文件识别规则

```python
# 测井文件：包含 'logging' 关键字，扩展名为.xlsx 或.csv
LOGGING_KW = ['logging']

# 表格文件：包含 'table' 或 'LITHO_TYPE' 关键字
TABLE_KW = ['table', 'LITHO_TYPE']

# FMI 文件：包含 'DYNA' 或 'STAT' 关键字，扩展名为.txt

# NMR 文件：包含 'nmr' 关键字，扩展名为.csv
```

```python
class DATA_WELL:
```

## 四、数据类型与文件格式

### 4.1 常规测井数据 (DataLogging)

**文件格式：** CSV 或 XLSX

**命名规则：** `{井名}_logging_data.csv` 或包含 `logging` 关键字

**数据结构：**

| 列名 | 类型 | 说明 |
|------|------|------|
| DEPTH | float | 深度 (m) |
| GR | float | 自然伽马 (API) |
| AC | float | 声波时差 (μs/m) |
| DEN | float | 密度 (g/cm³) |
| CNL | float | 中子孔隙度 (%) |
| RT | float | 深侧向电阻率 (Ω·m) |
| ... | ... | 其他曲线 |

**示例文件：** `SIMU4_logging_data.csv`

---

### 4.2 电成像数据 (DataFMI)

**文件格式：** TXT

**命名规则：** `{井名}-DYNA.txt` (动态) 或 `{井名}-STAT.txt` (静态)

**数据结构：**

- 每行代表一个深度点，每行第一个数据为深度指示数据
- 每列代表一个极板/按钮
- 值为电阻率或电导率
- 默认跳过文件头前八行

**示例文件：** `simu4_DYNA.txt`, `simu4_STAT.txt`

**纹理数据格式：**

| 列名 | 类型 | 说明 |
|------|------|------|
| DEPTH | float | 深度 (m) |
| CON_MEAN_STAT | float | 对比度均值 (静态) |
| DIS_MEAN_STAT | float | 差异性均值 (静态) |
| HOM_MEAN_STAT | float | 均匀度均值 (静态) |
| ... | ... | 其他纹理特征 |

**示例文件：** `SIMU4_texture_logging_120.csv`

---

### 4.3 表格数据 (DataTable)

**文件格式：** CSV 或 XLSX

**命名规则：** `{井名}__LITHO_TYPE.csv` 或包含 `LITHO_TYPE` 关键字

注意点：会默认生成一个类别replace规则，把type列的所有类别进行统计，然后映射成int类型数据，这个是为了可视化或者是后续的数据计算用的

**Type 3 格式 (深度区间)：**

| depth_start | depth_end | type |
|-------------|-----------|------|
| 3200.5 | 3210.2 | 砂岩 |
| 3210.2 | 3225.0 | 泥岩 |
| 3225.0 | 3230.5 | 砂砾岩 |

**Type 2 格式 (单深度点)：**

| depth | type |
|-------|------|
| 3200.0 | 砂岩 |
| 3201.0 | 砂岩 |
| 3202.0 | 泥岩 |

**示例文件：** `SIMU4__LITHO_TYPE.csv`

---

### 4.4 核磁数据 (DataNMR)

**文件格式：** CSV

**命名规则：** 包含 `nmr` 关键字

**数据结构：**

| 列名 | 类型 | 说明 |
|------|------|------|
| DEPTH | float | 深度 (m) |
| T2_0.1 | float | T2 谱 0.1ms 处振幅 |
| T2_0.2 | float | T2 谱 0.2ms 处振幅 |
| ... | ... | ... |
| T2_1000 | float | T2 谱 1000ms 处振幅 |

---

### 4.5 文件识别流程

```
1. 扫描井文件夹
   ↓
2. 遍历所有文件
   ↓
3. 检查文件名是否包含关键字
   ↓
4. 检查文件扩展名
   ↓
5. 分类到对应类型列表
   ↓
6. 返回路径列表
```

**搜索函数：** `search_files_by_criteria()`

**参数：**

- `name_keywords`: 名称关键字列表
- `file_extensions`: 扩展名列表
- `all_keywords`: 是否需要包含所有关键字

```
└─────────────────────────────────────────────────────────────┘
```

### 5.2 渲染流程

```
1. 数据管理器验证配置
   ↓
2. 计算子图数量
   ↓
3. 创建图形布局
   ↓
4. 设置滑动条控件
   ↓
5. 绘制各类型数据
   ├─ 测井曲线 (折线图)
   ├─ 分类数据 (填充图)
   ├─ FMI 图像 (热力图)
   └─ NMR 谱 (填充折线图)
   ↓
6. 设置交互事件
   ├─ 滑动条事件
   └─ 鼠标滚轮事件
   ↓
7. 显示图形
```

| 曲线开关 | 图例点击 | 显示/隐藏特定曲线 |
| 深度指示 | 悬停 | 显示当前深度值 |
